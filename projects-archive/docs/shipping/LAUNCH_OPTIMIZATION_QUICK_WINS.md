# Launch Optimization Quick Wins — 15 Minutes

## Reddit Launch Optimization

### Best Posting Times (Research-Based)

**r/webdev:**
- **Best:** Tuesday-Thursday, 9-11 AM EST
- **Why:** US + EU overlap, high engagement hours
- **Avoid:** Friday afternoon (weekend mode), Monday morning (busy)

**r/SaaS:**
- **Best:** Tuesday-Wednesday, 10 AM - 2 PM EST
- **Why:** Mid-week focus, founders actively browsing
- **Avoid:** Weekend (low activity)

**r/SideProject:**
- **Best:** Saturday-Sunday, 10 AM - 12 PM EST
- **Why:** Weekend builders, hobbyist peak time
- **Avoid:** Monday-Friday 9-5 (working hours)

### Title Optimization Formula

**Pattern:** [Built/Ship/Reject] + [Time Frame] + [Problem Solved] + [Unique Twist]

**Examples:**
- ❌ "My webhook logger app"
- ✅ "Built a webhook logger in 2 hours that needs no backend"
- ❌ "SaaS launch checklist"
- ✅ "Shipped in 2 hours: Webhook logger (localStorage only, zero cost)"

**Why works:**
- Specific time frame creates curiosity
- Technical details signal credibility
- Zero cost/backend = differentiation

### First Comment Strategy

**Immediate first comment (within 1 minute of posting):**

```
**Why I built this:**
- Existing tools are too complex (Hookdeck, Svix)
- I just needed to paste JSON and save it
- localStorage = zero infrastructure, zero cost

**Tech stack:**
- Pure HTML/CSS/JS (no frameworks)
- 620 lines, 20KB total
- GitHub Pages hosting (free)

**Feedback welcome:**
- Would you use this?
- What features are missing?
- Should I add X or keep it simple?
```

**Why works:**
- Shows authenticity (not spam)
- Pre-answers common questions
- Encourages engagement (questions)
- Demonstrates technical competence

## Product Hunt Launch Optimization

### Best Launch Times

**Data from 1000+ launches:**
- **Best:** Tuesday 12:01 AM PST (3:01 AM EST)
- **Second:** Wednesday 12:01 AM PST
- **Why:** Fresh day for hunters, algorithm boost
- **Avoid:** Friday (weekend drop), Monday (catch-up mode)

### First Hour Strategy (Critical Window)

**0-15 minutes:**
- Respond to every comment immediately
- Upvote every comment
- Answer questions with specificity

**15-60 minutes:**
- Share on Twitter/LinkedIn
- Ask friends to upvote (not downvote)
- Post in Product Hunt Slack/Discord

**Comment template:**
```
Thanks [Name]! 

Yes, I built this because [specific pain point].

The tech stack is [simple explanation] - intentionally minimal to solve [problem].

What would make this more useful for you?
```

**Why works:**
- Personal responses = algorithm boost
- Asking questions = more comments
- Specificity = credibility

### Community Amplification

**Discord servers to post in (launch day):**
1. Product Hunt (official)
2. Indie Hackers
3. No-Code Founders
4. Maker's Lane
5. Side Project Builders

**Message template:**
```
Hey! Just launched [Product Name] on Product Hunt 🔥

[One sentence what it does]
[Link]

Would love your feedback - what do you think?
```

**Why works:**
- Personal message (not mass blast)
- Specific ask (feedback, not just upvote)
- Emoji = friendly tone

## Social Amplification Optimization

### Twitter Best Practices

**Posting timing:**
- **Best:** 9-11 AM EST or 2-4 PM EST
- **Why:** Commute + lunch breaks
- **Frequency:** 3-4 posts on launch day (8 AM, 12 PM, 4 PM, 8 PM EST)

**Thread structure:**
1. **Hook tweet:** "Built X in Y hours because Z"
2. **Problem:** "Existing tools are [pain point]"
3. **Solution:** "So I built [minimal solution]"
4. **Tech:** "Stack: [simple], Lines: [small], Cost: [zero]"
5. **Launch:** "Live at [link], feedback welcome"
6. **Tag:** #ProductHunt #[Category]

**Example:**
```
🧵 1/6

Built a webhook logger in 2 hours.

Why? Because existing tools are too complex.

I just needed to paste JSON and save it - not configure a full monitoring platform.
```

**Why works:**
- Thread format = algorithm boost
- Numbers (2 hours) = curiosity
- Thread icon = visual signal

### LinkedIn Best Practices

**Posting timing:**
- **Best:** Tuesday-Thursday, 10-11 AM EST or 2-3 PM EST
- **Why:** Professional hours, mobile commute
- **Length:** 1000-1500 characters (not too short, not too long)

**Structure:**
```
[Hook] Built a webhook logger in 2 hours. Here's why:

[Context] I was debugging webhook integrations and needed a simple way to inspect JSON payloads. Existing tools like Hookdeck and Svix are powerful but overkill for quick debugging.

[Solution] So I built a minimal logger:
- Paste JSON → Save → Pretty print
- No backend (localStorage only)
- Zero infrastructure cost
- 620 lines of pure HTML/CSS/JS

[Philosophy] Ship simple > build complex. DHH approach: 2-hour MVP > 2-week platform.

[Launch] Live on GitHub: [link]

Would love feedback from other developers - is this useful or too simple?
```

**Why works:**
- Professional tone
- Philosophy shows depth
- Specific feedback ask

## Quick Wins Summary

### Before You Post (2 minutes)
1. Check Reddit/subreddit active hours
2. Prepare first comment (copy-paste ready)
3. Have GitHub/Product Hunt links in clipboard
4. Screenshot ready (if applicable)

### After You Post (3 minutes per platform)
1. Post content
2. Immediately post first comment
3. Upvote/respond to first comments within 5 minutes
4. Monitor for 1 hour, respond to all comments

### Day-Of Checklist
- [ ] Reddit posts scheduled (9-11 AM EST)
- [ ] First comment prepared for each post
- [ ] Product Hunt time confirmed (12:01 AM PST)
- [ ] Twitter thread drafted
- [ ] LinkedIn post drafted
- [ ] Discord messages prepared
- [ ] Analytics setup (GitHub, Product Hunt, Google Analytics)

### Engagement Rules
- Reply to ALL comments within 1 hour
- Upvote helpful responses
- Ask specific questions (not "thoughts?")
- Share technical details (builds credibility)
- Be authentic, not promotional
