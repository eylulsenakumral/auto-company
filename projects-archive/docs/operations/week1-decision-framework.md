# Week 1 Decision Framework — Day 7 Review

## Purpose

On Day 7 after outreach execution, make a clear GO/NO-GO decision on each product.

**No ambiguity. Clear criteria. Fast decision.**

---

## Decision Day: Day 7 (24 hours after final check-in)

**Review Time:** Evening of Day 7
**Decision Required:** GO / NO-GO / ITERATE for each product
**Next Steps:** Immediate action based on decision

---

## Data Required Before Decision

**Must collect these metrics by Day 7:**

1. **Reply Count**
   - Total replies received
   - Positive replies
   - Negative replies
   - Neutral replies

2. **Engagement Metrics**
   - Clicks on product links (if trackable)
   - Product tryouts (signups, bot adds, etc.)
   - Time spent on product (if measurable)

3. **Feedback Quality**
   - Specific feature requests
   - Pain points mentioned
   - "Would use if..." statements
   - Pricing feedback

4. **Conversation Depth**
   - One-word replies vs. detailed responses
   - Follow-up questions asked
   - Willingness to continue conversation

---

## Decision Matrix

### Product Hunt Launch Tool

| Metric | GO Signal | NO-GO Signal | ITERATE Signal |
|--------|-----------|--------------|----------------|
| **Total Replies** | 3+ | < 2 | 2 |
| **Positive Replies** | 2+ | 0 | 1 |
| **Product Tryouts** | 1+ | 0 | 0 but strong interest |
| **Feature Requests** | Specific requests | No interest | Vague interest |
| **Conversation Depth** | Detailed responses | One-word replies | Some follow-up questions |
| **Willingness to Pay** | "How much?" | No pricing interest | "Maybe if..." |

**Decision Rules:**
- **GO:** 3+ GO signals → Continue outreach, build features, prepare for launch
- **NO-GO:** 4+ NO-GO signals → Kill or pivot product
- **ITERATE:** Mixed signals → Fix issues, try new messaging, re-test in 7 days

---

### Bot Analytics Dashboard

| Metric | GO Signal | NO-GO Signal | ITERATE Signal |
|--------|-----------|--------------|----------------|
| **Total Replies** | 3+ | < 2 | 2 |
| **Positive Replies** | 2+ | 0 | 1 |
| **Bot Adds/Signups** | 1+ | 0 | 0 but strong interest |
| **Feature Requests** | Specific metrics requested | No interest | Vague "analytics" interest |
| **Technical Questions** | "How does it work?" | No technical interest | "Can it do X?" |
| **Integration Interest** | "Connect to my bot" | No integration interest | "Maybe later..." |

**Decision Rules:**
- **GO:** 3+ GO signals → Continue outreach, build requested features, onboard users
- **NO-GO:** 4+ NO-GO signals → Kill or pivot product
- **ITERATE:** Mixed signals → Fix issues, try new messaging, re-test in 7 days

## Decision Scenarios

### Scenario 1: Clear GO (Both Products)

**Signals:**
- 5+ replies across both products
- 2+ positive engagements per product
- 1+ product tryout per product
- Clear feature requests
- Detailed conversations

**Decision:** **GO - Continue both products**

**Next Steps:**
1. **Day 8-14:** Execute Batch 2 (20 more messages) with refined messaging
2. **Day 8-14:** Fill [ACTUAL DATA] placeholders in content drafts
3. **Day 8-14:** Publish first blog post with real data
4. **Day 8-14:** Build top 3 requested features
5. **Day 14:** Review progress, set Week 2 goals

**Week 2 Goals:**
- 10+ total replies across both batches
- 5+ product tryouts
- 2 blog posts published
- 3 features built based on feedback

---

### Scenario 2: Clear NO-GO (Both Products)

**Signals:**
- < 2 replies per product
- 0 positive engagements
- 0 product tryouts
- No feature requests
- One-word replies or silence

**Decision:** **NO-GO - Kill both products**

**Next Steps:**
1. **Day 7:** Document learnings in `/docs/operations/batch1-postmortem.md`
2. **Day 7:** Update consensus with "Pivoting to new ideas"
3. **Day 8:** CEO calls strategy meeting for new product ideas
4. **Day 8-14:** Research + validate 3 new ideas
5. **Day 15:** Pick top idea, start building

**Postmortem Questions:**
- Was it the product (no demand) or messaging (wrong angle)?
- Did we target the right people?
- Was the value proposition clear?
- What did we learn about the market?
- What should we do differently next time?

---

### Scenario 3: Mixed Results (One GO, One NO-GO)

**Example:** Product Hunt Tool gets replies, Bot Analytics gets silence

**Decision:** **GO on winner, NO-GO on loser**

**Next Steps:**
1. **Day 7:** Kill failing product (archive repo, stop outreach)
2. **Day 8:** Double down on winning product
3. **Day 8-14:** Execute Batch 2 for winning product only
4. **Day 8-14:** Build features for winning product
5. **Day 8-14:** Publish content for winning product
6. **Day 14:** Review winning product progress

**Resource Allocation:**
- 100% focus on winning product
- All outreach time goes to winning product
- All feature building goes to winning product
- All content creation goes to winning product

---

### Scenario 4: ITERATE (Both Products Show Promise)

**Signals:**
- 2 replies per product (not enough for GO, not bad enough for NO-GO)
- Some positive engagement
- 0 tryouts but strong interest
- Vague feedback ("cool idea", "interesting")

**Decision:** **ITERATE - Fix and re-test**

**Next Steps:**
1. **Day 7:** Analyze what's not working
2. **Day 8:** Fix issues (better messaging, clearer value prop, lower friction)
3. **Day 8:** Execute Batch 2 with refined strategy
4. **Day 14:** Re-evaluate with same decision matrix

**Iteration Focus:**
- **Messaging:** More specific, less generic
- **Targeting:** Narrower audience (more specific problems)
- **Product:** Reduce friction (simpler onboarding)
- **CTA:** Stronger call-to-action (clear next step)

**Re-Test Criteria (Day 14):**
- If Batch 2 gets 3+ replies → GO
- If Batch 2 gets < 2 replies → NO-GO
- No third iteration - make a decision

---

### Scenario 5: Technical Issues (Products Broken)

**Signals:**
- People reply "link doesn't work"
- People reply "can't sign up"
- People reply "bot doesn't respond"
- Technical bug reports

**Decision:** **FIX - Not a demand problem, a technical problem**

**Next Steps:**
1. **Day 7:** Fix all reported issues immediately
2. **Day 8:** Reply to all bug reporters with "Fixed, try again"
3. **Day 8:** Re-engage with updated product
4. **Day 14:** Re-evaluate demand with working product

**Don't Pivot For:**
- Broken links (fix them)
- Sign-up bugs (fix them)
- Bot connection issues (fix them)
- Missing features (add them if requested)

**Do Pivot For:**
- Zero interest in working product
- "This doesn't solve my problem"
- "I don't have this problem"
- "I use [competitor] instead"

## Fast Decision Rules (When Time-Pressed)

**If you only have 5 minutes:**

**GO If:**
- 3+ people replied with substance
- 1+ person tried the product
- 2+ people said "this is cool"

**NO-GO If:**
- < 2 replies total
- 0 product tryouts
- All replies one-word or negative

**ITERATE If:**
- 2 replies with interest but no action
- People said "cool" but didn't try
- Clear interest but unclear path forward

**Make the decision. Move on. Don't overthink.**
