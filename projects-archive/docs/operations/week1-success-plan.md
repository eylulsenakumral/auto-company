# Week 1 Success Plan — Day 4-7

## Purpose
Clear plan for Week 1 (Day 4-7) after outreach execution. What to do, when to do it, how to measure success.

## Week 1 Goals

**Primary Goal:** Get first users and learn from them

**Success Metrics:**
- Replies received: 5+ (25% response rate from 20 messages)
- Positive engagements: 4+ (80% of replies)
- Product tryouts: 2+ (50% of positive replies)
- Feedback captured: 100% of tryouts
- Features prioritized: Top 5 requests identified

**Minimum Viable Success (MVS):**
- 3+ replies total
- 1+ person tries product
- 1+ piece of useful feedback

**Failure Point (Trigger Batch 2):**
- < 3 replies total
- All replies negative
- Zero product tryouts

### Decision Framework (Day 7 afternoon)

**Trigger Batch 2 IF:**
- Total replies < 3 (15%)
- Positive replies < 2 (10%)
- Zero product tryouts
- Clear signal: messaging/channel mismatch

**Continue Product Iteration IF:**
- 5+ replies (25%+)
- 2+ product tryouts
- Clear feature requests
- Valid feedback received

**Pivot/Kill Product IF:**
- 3+ replies, all negative
- "Not useful" feedback from multiple people
- Clear signal: problem doesn't exist
- Clear signal: solution doesn't fit

## Week 2 Scenarios

### Scenario A: Success (5+ replies, 2+ tryouts, good feedback)

**Week 2 Actions:**
1. Build top 3 requested features
2. Fix top 2 bugs/UX issues
3. Send Batch 2 to 20 more targets (with improved messaging)
4. Create content post about learnings
5. Write blog post: "What I learned from first 10 users"

**Week 2 Goals:**
- 10+ total users
- 5+ active users (used product 2+ times)
- Clear picture of PMF (or not)
- 20+ outreach messages sent (Batch 2)

### Scenario B: Mixed Response (3-4 replies, 1 tryout, some positive/some negative)

**Week 2 Actions:**
1. Analyze what worked vs what didn't
2. Improve messaging based on learnings
3. Send Batch 2 (20 more targets)
4. Fix critical issues mentioned in feedback
5. Try different channels (Twitter, LinkedIn)

**Week 2 Goals:**
- Better response rate (30%+)
- Clearer picture of what works
- 3+ total users

### Scenario C: Failure (< 3 replies, zero tryouts)

**Week 2 Actions:**
1. Trigger Batch 2 immediately (Day 8)
2. Execute with refined messaging
3. If Batch 2 also fails → Pivot/Kill
4. Document learnings in "post-mortem"
5. Move to next product idea

**Week 2 Goals:**
- Batch 2 sent (20 messages)
- Learn if product has demand
- If no demand → Kill quickly, move on
- If some demand → Iterate
