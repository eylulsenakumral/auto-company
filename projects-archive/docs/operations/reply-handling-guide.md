# Reply Handling Guide — Day 4-7

## Core Principles

1. **Reply Fast:** Within 1 hour for first 24 hours
2. **Be Authentic:** Not promotional, conversational
3. **Add Value:** Every reply should help them
4. **Ask for Feedback:** Not "buy my thing", just feedback
5. **Build Relationships:** Long-term > short-term conversion

---

## Reply Decision Tree

### Is the response positive or neutral?
**YES → Engage deeper**
- Thank them
- Ask specific questions
- Offer to DM for more detailed discussion
- Invite to try product + send feedback

**NO (Negative/Spam accusation) → De-escalate**
- Apologize if intrusion
- Clarify intent (helpful, not promotional)
- Exit gracefully
- Don't argue

### Is it a question?
**YES → Answer directly + offer more help**
- Direct answer first
- Link to relevant feature/docs
- Offer to continue discussion

**NO (Comment/Opinion) → Acknowledge + redirect**
- Acknowledge their point
- Share relevant perspective
- Ask what they think

### Is it feedback?
**YES → Capture + Iterate**
- Thank them profusely
- Ask follow-up questions
- Tell them you'll work on it
- Offer to update them

## Reply Templates (By Response Type)

### Type 1: "What is this?" / "Why did you message me?"

```
Hey! Great question. 

I saw your post about [their topic] and it seemed like you're working on [specific challenge]. I built a free tool that helps with exactly this, so I thought it might be useful for you.

No pressure at all - just trying to be helpful to the community. If it's not relevant, no worries!

Happy to answer any questions about it.
```

### Type 2: "This looks cool" / "Interesting"

```
Thanks! If you get a chance to try it out, I'd love to know what you think.

I built it specifically for [their use case], so any feedback on what would make it more useful for you would be super helpful.

Link: [PRODUCT_LINK] if you want to check it out.
```

### Type 3: "I'll try it later" / "Maybe"

```
Awesome, no rush! 

If you do try it, I'd be curious to hear what you think - even if it's just "this is confusing" or "needs X feature".

Building this for people like [their situation], so real feedback is gold for me.
```

### Type 4: Specific Question

```
Great question! 

[Direct answer to their question]

[If relevant] You can see this in action here: [SPECIFIC_LINK]

Happy to explain more if you're interested - this is exactly the kind of thing I'm trying to get right.
```

### Type 5: Feedback / Suggestion

```
This is exactly the kind of feedback I'm looking for - thank you!

[Specific acknowledgment: "That's a great point about X" or "I didn't think about Y"]

I'm going to work on [improvement based on their feedback]. Would you be open to me messaging you when I've updated it? Would love to get your thoughts on the changes.

Your feedback just made this better for everyone 👍
```

### Type 6: "This is spam" / "Don't message me"

```
Hey, I apologize if this came across as spam or intrusive - that definitely wasn't my intent.

I saw your post about [topic] and genuinely thought my tool might be helpful for the challenge you're working on. I'm not trying to sell anything - just looking for feedback from people dealing with real problems.

I won't message you again. Best of luck with [their project]!
```

### Type 7: "How does this compare to X?"

```
Great question! 

[Honest comparison - acknowledge X's strengths, highlight where your tool is different]

I built this because [specific gap/need]. For [their use case], I think it helps because [specific benefit].

But if [X tool] is working well for you, stick with it! Just another option if you ever need it.
```

### Type 8: "Can you add X feature?"

```
Yes! That's actually on my roadmap, but you're the first person to ask for it - which tells me it's important.

Here's my current plan: [rough timeline/idea]

Would you be open to me DMing you when I've added it? Would love to get your feedback on whether it solves your problem.
```

### Type 9: "This doesn't work for me"

```
Oh no, sorry about that! Let me help you troubleshoot.

[Ask specific questions]:
- What were you trying to do?
- What happened when you tried?
- Can you share a screenshot or more details?

I want to make sure this works for everyone, so I'm happy to dig into this and fix it.
```

### Type 10: "Who are you?" / "Why did you build this?"

```
Great story! 

[Short, honest version: "I'm a builder who saw X problem and wanted to solve it"]

I built this because [specific frustration/pain point]. Couldn't find anything that solved it the way I wanted, so I built it.

Now I'm just trying to find out if it's actually useful for other people or if I'm solving a problem nobody has 😅

Happy to share more about the journey if you're interested!
```

## Engagement Flow

### First Reply (Within 1 hour)
- Use template above
- Be conversational, not transactional
- End with question (keeps conversation going)
- Don't ask for sale, ask for feedback

### Second Reply (If they engage)
- Deeper discussion about their use case
- Share specific features that help
- Offer to DM for detailed discussion
- Build relationship, not conversion

### Third Reply (Deep engagement)
- Invite to beta testing (if relevant)
- Ask for detailed feedback session
- Offer early access to features
- Make them feel like insider/contributor

```markdown
## Feedback #[Number]

**Date:** [Timestamp]

## Success Metrics

**Week 1 Goals:**
- Replies received: _____ / 20 (aim for 5+ = 25%)
- Positive engagements: _____ / replies (aim for 80%+)
- Product tryouts: _____ / replies (aim for 50%+)
- Feedback captured: _____ / tryouts (aim for 100%)
- Features prioritized: _____ top requests identified

**What Success Looks Like:**
- 5+ people reply with thoughtful responses
- 2+ people actually try the product
- 1+ person gives detailed feedback
- Clear picture of what to build next

**What "Failure" Looks Like (and what to do):**
- 0-1 replies → Messaging wrong, improve and resend to new 20
- 2-3 replies, all negative → Product concept wrong, pivot or kill
- Replies but no tryouts → Value prop unclear, fix landing page
- Tryouts but no feedback → Onboarding broken, improve UX

- [ ] Did I reply within 1 hour? (first 24 hours)

### They say "I use X and it's great"

```
That's awesome! 

[If X is competitor]: Yeah, [X] is solid. I built this because [specific gap]. But if [X] is working for you, definitely stick with it!

If you ever try mine, I'd be curious to hear how they compare - always trying to learn from other approaches.
```

### They ask "How much does it cost?"

If you try it and find it useful, the only "payment" I'm asking for is your honest feedback on what could make it better 😅
```

### They ask "Is this open source?"

```

1. Transparency - you can see exactly what it does
2. Learning - others can see how it's built
3. Trust - no hidden data collection

If you're a developer, happy to have contributions! Or if you spot any issues, PRs are welcome 😅
```

## When to Stop Replying

**Stop if:**
- They haven't replied in 48 hours (send one follow-up, then stop)
- They say "not interested" (thank them, exit gracefully)
- Conversation becomes circular/polite (thank, offer future help, stop)
- They ask you to stop (apologize, stop immediately)

**Keep going if:**
- They're asking detailed questions
- They're giving feedback
- They seem genuinely interested
- They've tried the product and have questions

---

## Red Flags (Warning Signs)

⚠️ **Be careful if:**
- They ask for personal info (email, phone, etc.)
- They want to move to another platform immediately
- They offer to "help" in vague ways
- They ask for money/investment
- They seem too good to be true

**Response to red flags:**
```
Thanks for the interest! 

I'm keeping all discussion here on [platform] for transparency. If you have questions about the product, happy to answer - but I'm not looking for investment/money/help right now. Just trying to build something useful 😅
```
