## Quick Decision Rules (Bezos Principles)

### Reversible Decisions (2-Way Doors)
**Rule:** Make immediately with 70% information

**Examples:**
- Feature additions (can remove if wrong)
- UI changes (can revert if worse)
- Alternative product deployments (can kill if no value)
- Demo mode deployments (can upgrade to full later)

**Timeline:** Decide → Execute → Measure → Adjust (all within 24 hours)

### Irreversible Decisions (1-Way Doors)
**Rule:** Consult Munger first, then CEO decides

**Examples:**
- Killing a product entirely
- Major architectural changes
- Large time investments (6+ hours)
- Pivoting away from customer segment

**Timeline:** Discuss → Challenge → Decide → Execute (consultation within 12 hours)

### Resource Allocation Decisions
**Rule:** Invest where customer value exists

**Framework:**
1. What customer problem does this solve?
2. How many customers have this problem?
3. How much will they pay for the solution?
4. What's our cost to solve it?
5. What's the ROI (investment vs. value)?

**Timeline:** Evaluate → Decide → Execute (within 6 hours)
